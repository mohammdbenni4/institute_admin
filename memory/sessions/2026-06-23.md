## Decisions Made
- **Revision scoring** is all-or-nothing: all parts succeeded → full points (rating 4); any fail → 0 (rating 1).
- Revision rows serialize into the record's `revision_lesson` as an Arabic message, e.g.
  `الجزء ٢٩ (النصف الأول): نجح، الجزء ٣٠ (النصف الثاني): أخفق`.
- Excel import: placeholder "—" for missing father fields; create-missing-halaqahs uses first teacher+type
  as placeholders (admin edits later); unmatched halaqahs block until created/ignored.
- Scoring weights may be negative (penalties allowed).
- Mobile problems picker lives on the التسميع والمراجعة page (not الحضور).

## Key Learnings
- **API list endpoints cap `limit` at 200** (daily-records, students) — fetch large sets by paging offsets.
  Bit us twice (admin attendance, mobile heatmap). `/problems` allows up to 500.
- Server has **no `uv`**; backend runs from a pip `.venv` (`backend/.venv/bin/alembic|uvicorn`) via systemd.
- SvelteKit `$env/dynamic/public` is read at runtime → PM2 `env` provides `PUBLIC_API_BASE_URL` on restart;
  no rebuild needed for that value.
- Svelte 5: `$derived.by(() => …)` for computed maps (not `$derived(() => …)`).
