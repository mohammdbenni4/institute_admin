## Decisions Made
- "Best student" metric = sum of total_points for the selected month (user choice).
- Card scores stored as a snapshot of the scoring policy at write time — changing weights never rewrites history.
- Teacher cross-tenant access returns 404 (hide existence) for reads, 403 for writes.
- Match the Stitch design's *system*, not pixel-exact (user direction).
- Mobile is a separate standalone SvelteKit project (port 5174) reusing the admin's API-client patterns.

## Key Learnings
- FastAPI yield-dependency teardown (session commit) runs after the response — caused a one-off read-after-delete flake in an e2e (not a real bug).

- The admin API client lacked `put`; added it for the scoring PUT endpoint.
- Backend must run with `export PYTHONPATH=` to avoid global ROS Jazzy pollution.
